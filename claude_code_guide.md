# 🤖 Claude Code 開発ガイドライン (Musuhi プロジェクト)

本ドキュメントでは、AnthropicのCLIツール「Claude Code」を活用して、本プロジェクト（Musuhi）の開発を効率的に進めるためのワークフローやプロンプト例をまとめています。

## 📁 プロジェクト構成と前提知識

本プロジェクトは主に以下の2つのディレクトリに分かれています。Claude Codeに指示を出す際は、どちらのシステムに手を加えるのかを明確にすると精度が上がります。

- **`Musuhi-Web/`** (Next.js フロントエンド)
  - UIコンポーネント、ページルーティング、フロントエンドの状態管理、APIとの通信処理など。
- **`musuhi-ai-api/`** (FastAPI バックエンド)
  - AI関連の処理、Dify APIとの連携、バックエンドのロジックなど。

## 🔄 基本的な開発ワークフロー

Claude Codeを使った開発は、**「Issueドリブン」**で進めることを推奨します。以下のステップで作業を行います。

### Step 1: Issueの作成
実装したい機能や修正内容が決まったら、まずGitHubにIssueを作成します。これもClaude Codeに行わせることが可能です。

```bash
# プロジェクトのルートディレクトリで実行
claude
```
> **プロンプト例:**
> 「『○○機能の追加』というタイトルで、GitHubにIssueを作成してください。要件は以下の通りです：...」

### Step 2: feature ブランチの作成
作成したIssue番号をもとに、作業用のfeatureブランチを作成します。

> **プロンプト例:**
> 「先ほど作成したIssue #10 に対応するための feature/issue-10-xxx ブランチを作成し、チェックアウトしてください。」

### Step 3: Claude Code による実装作業
要件を元に、Claude Codeに実装を依頼します。一気に全てを作らせるよりも、ステップを分けて確認しながら進めるのがコツです。

> **プロンプト例:**
> 「Issue #10 の要件に従って実装を進めてください。まずは `Musuhi-Web/src/app/` 以下の対象ページコンポーネントの構造を作成し、完了したら教えてください。」

### Step 4: 動作確認と修正
実装が完了したら、ローカルサーバーを立ち上げて動作確認を行います。エラーが出た場合は、エラーログをそのままClaude Codeに渡して修正させます。

> **プロンプト例:**
> 「Next.jsでビルドエラーが出ました。以下のエラーログを確認して、該当箇所を修正してください。
> [エラーログをペースト]」

### Step 5: コミットとPull Requestの作成
作業が完了したら、コミットしてPull Requestを作成させます。

> **プロンプト例:**
> 「現在の変更をコミットし、GitHubにプッシュしてください。その後、Issue #10 をクローズするためのPull Requestを作成してください。」

## 💡 Claude Codeを活用するためのプロンプトのコツ

Claude Code（エージェント）に適切に動いてもらうためには、文脈と制約をはっきり伝えることが重要です。

### 1. 範囲を限定する
プロジェクト全体を見に行かせると時間がかかるため、対象となるディレクトリやファイルを絞って指示します。
> ⭕️ **良い例:** 「`Musuhi-Web/src/components/` 内にあるボタンコンポーネントのスタイルを修正して。」
> ❌ **悪い例:** 「全体のボタンのデザインを直して。」

### 2. 段階的に進める（Plan & Execute）
複雑な実装の場合は、いきなりコードを書かせるのではなく、まず設計や方針を出させます。
> **プロンプト例:**
> 「FastAPI (`musuhi-ai-api/`) に新しいエンドポイントを追加したいです。コードを書く前に、どのようなファイル構成・URL設計にするか計画を提示してください。」

### 3. コマンドの実行許可について
Claude Codeはファイルの読み書きだけでなく、ターミナルコマンドの実行（`npm run dev` や `git commit` など）も行えます。実行を求められたら内容を確認し、問題なければ許可（`y` または `Enter`）してください。

### 4. 既存のコードスタイルに合わせるよう指示する
> **プロンプト例:**
> 「新しいコンポーネントを作成してください。ただし、既存の `Musuhi-Web/src/components/` の実装スタイル（Tailwindの書き方や型定義の方法）を踏襲してください。」

## ⚠️ 注意点
- Claude Codeは非常に強力ですが、たまに不要なファイルを消したり、意図しないコードの書き換えを行うことがあります。**常にGitの変更差分（`git diff`）を確認しながら**進めてください。
- バックエンド (`musuhi-ai-api`) とフロントエンド (`Musuhi-Web`) の両方をまたぐ修正の際は、どちらのサーバーを立ち上げてテストすべきかを明確に指示するとスムーズです。
