# issue-new — GitHub Issue 対話作成スキル

ユーザーとの対話を通じて、プロジェクトのIssueテンプレートに沿った質の高いIssueを作成する。
各ステップでユーザーの回答を待ってから次へ進むこと。

## フロー概要

1. Issueの種類を確認する
2. タイトルと詳細をヒヤリングする
3. プレビューを表示してユーザーの確認を取る
4. `gh issue create` でGitHubに作成する

---

## Step 1: 種類を確認する

以下のメッセージを表示してIssueの種類を聞く：

```
📝 GitHub Issue 作成アシスタントです。

どのような内容のIssueを作成しますか？

1. 🆕 機能追加 — 新しい機能を追加したい
2. 🐛 バグ修正 — 不具合・エラーを報告したい
3. ⚡ 改善・リファクタリング — 既存機能を改善したい
4. 📚 ドキュメント — ドキュメントを追加・修正したい
5. 💡 その他

番号または種類名で答えてください。
```

---

## Step 2: タイトルを収集する

```
【Q1】タイトルを教えてください。
Issueの内容を一言で表した短いフレーズでお願いします。
（例: 「ボイスギフト送信後に通知が届かない」「お気に入り機能を追加したい」）
```

---

## Step 3: 詳細をヒヤリングする（種類に応じて変える）

**機能追加・改善の場合:**
```
【Q2】実現したいことを詳しく教えてください。
どのような機能で、ユーザーにどんな価値を提供しますか？

【Q3】その機能が必要な背景・理由は何ですか？
```

**バグ修正の場合:**
```
【Q2】どのような状況でバグが発生しますか？
（操作手順・発生条件を具体的に教えてください）

【Q3】期待される動作と実際の動作を教えてください。
・期待: 
・実際: 
```

**ドキュメントの場合:**
```
【Q2】どのドキュメントを、どのように追加・修正したいですか？
```

---

## Step 4: 優先度と補足情報を収集する

```
【Q4】優先度はどのくらいですか？
・🔴 高（なるべく早く対応したい）
・🟡 中（余裕があるときに対応）
・🟢 低（いつかやりたい）

【Q5】実装のアイデアや参考情報があれば教えてください。（なければ「なし」でOK）
```

---

## Step 5: プレビューを表示して確認を取る

収集した情報を以下のフォーマットに整形してプレビューを表示する。

**タイトル形式:** `[絵文字] タイトル`（機能追加→🆕 バグ→🐛 改善→⚡ ドキュメント→📚 その他→💡）

**機能追加・改善の本文（`.github/ISSUE_TEMPLATE/機能開発---feature-request.md` 参照）:**
```markdown
## 概要
[実現したいことの概要]

## AS-IS (現在の状態)
[現在の状態・問題点]

## TO-BE (理想の状態)
[改善後にどうなっていてほしいか]

## GOAL (具体的なタスク・完了条件)
- [ ] 
- [ ] 

## 参考
[実装アイデアや参考URL]
```

**バグ修正の本文（`.github/ISSUE_TEMPLATE/bug-report.md` 参照）:**
```markdown
## 概要
[バグの概要]

## 再現手順
1. 
2. 

## 期待される動作
[本来どうなるべきか]

## 実際の動作
[現在どうなっているか]

## 補足情報
[スクリーンショット・ログ・環境情報など]

## 優先度
[高/中/低]
```

プレビュー表示後に確認を取る：
```
このIssueを作成してよいですか？
修正が必要であれば変更点を教えてください。（「OK」または修正内容）
```

---

## Step 6: GitHubにIssueを作成する

ユーザーが「OK」と答えたら：

```bash
# リポジトリ情報を確認
git remote get-url origin

# Issueを作成
gh issue create \
  --title "タイトル" \
  --body "$(cat <<'EOF'
本文
EOF
)" \
  --label "ラベル"
```

ラベルの対応:
- 機能追加・改善 → `enhancement`
- バグ修正 → `bug`
- ドキュメント → `documentation`
- その他 → ラベルなし

gh CLIが未認証の場合は以下を表示して終了:
```
⚠️ gh CLIが未認証またはインストールされていません。
以下を実行してセットアップしてください：
  brew install gh   # インストール
  gh auth login     # 認証
```

---

## Step 7: 完了メッセージ

```
✅ Issueを作成しました！
🔗 [Issue URL]
📌 Issue番号: #XX

次のステップ：
・新機能・改善を実装する場合 → /feature を実行
・緊急バグ修正の場合 → /hotfix を実行
```
